# Plan: Backend Research Persistence

Transition research session and event persistence from client-side IndexedDB to the backend libSQL database.

## 1. Backend Schema Changes

Create a new migration `pkg/db/002_research_persistence.up.sql`:

- **Table: `research_sessions`**
  - `id` TEXT PRIMARY KEY (UUID generated by frontend)
  - `codebase_id` INTEGER NOT NULL
  - `title` TEXT NOT NULL
  - `state` TEXT NOT NULL
  - `created_at` INTEGER NOT NULL
  - `archived_at` INTEGER (NULL if active)
  - `FOREIGN KEY(codebase_id) REFERENCES codemogger_codebases(id)`
- **Table: `research_events`**
  - `id` INTEGER PRIMARY KEY AUTOINCREMENT
  - `session_id` TEXT NOT NULL
  - `type` TEXT NOT NULL ('sse' or 'ce')
  - `timestamp` INTEGER NOT NULL
  - `data` TEXT NOT NULL (Raw JSON string)
  - `FOREIGN KEY(session_id) REFERENCES research_sessions(id) ON DELETE CASCADE`

## 2. Backend Store implementation

- `pkg/db/research_schema.go`: Contains `ResearchSession` and `ResearchEvent` structs.
- `pkg/db/research_store.go`:
  - `SaveResearchSession(session)`: Upserts session metadata.
  - `GetResearchSessionByCodebase(codebaseID)`: Finds the latest active session for a codebase.
  - `ListResearchSessions(includeArchived bool)`: Lists sessions for navigation.
  - `DeleteResearchSession(sessionID)`: Removes session and cascaded events.
  - `SaveResearchEvent(event)`: Appends a raw event to the session history.
  - `GetResearchEvents(sessionID)`: Retrieves all events for rehydration.
  - `PruneArchivedSessions(maxLimit)`: Deletes oldest archived sessions.

## 3. API Endpoints

- `GET /api/research/sessions`: Returns active research sessions for sidebar.
- `GET /api/research/sessions/{id}/events`: Returns raw event stream for rehydration.
- `POST /api/research/sessions`: Creates/Updates a session.
- `POST /api/research/sessions/{id}/archive`: Marks a session as archived.
- `DELETE /api/research/sessions/{id}`: Deletes a session.

### Automatic Event Logging
The `handleAgentResearch` handler in `pkg/server/api/api_agent.go` is updated to:
1.  Use `persistenceStreamWriter` to intercept `SSE` chunks and `CEEvent` emitted by the agent.
2.  Persist them to the `research_events` table before flushing to the client.
